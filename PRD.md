## 1. 개요

### 1.1 목적

Microsoft Agent Framework(MAF)와 OpenAI API를 기반으로, 날씨·골프장 예약·개인 일정을 통합 조회할 수 있는 멀티에이전트 MVP를 구축한다. 사용자는 채팅 UI 하나로 세 도메인의 정보를 카드 형태로 받아볼 수 있다.

### 1.2 범위

| 포함 | 제외 |
|------|------|
| SuperAgent + 3개 SubAgent 파이프라인 | 실제 예약 트랜잭션 |
| 채팅 UI (카드 응답) | 사용자 인증/권한 관리 |
| Schedule Sub의 Mock MSA | 실 캘린더 연동 (Google/Outlook) |
| 수도권 골프장 스크래핑/공개 API | 골프장 결제 연동 |

## 4. 에이전트 상세 명세

### 4.1 SuperAgent

| 항목 | 내용 |
|------|------|
| 역할 | 사용자 메시지를 분석해 SubAgent를 선택·호출하고 결과를 통합 응답 |
| LLM | OpenAI `gpt-4.5` (기본값, 환경변수로 오버라이드 가능) |
| 입력 | 사용자 자연어 메시지 |
| 출력 | 카드 + 텍스트 혼합 응답 |
| 라우팅 규칙 | 날씨 키워드 → Sub#1, 골프 키워드 → Sub#2, 일정/스케줄 키워드 → Sub#3, 복합 의도 → 복수 Sub 병렬 호출 |
| Fallback | 어떤 Sub에도 해당하지 않으면 일반 LLM 답변 반환 |

### 4.2 SubAgent#1 — Weather

| 항목 | 내용 |
|------|------|
| 역할 | 특정 월의 일자별 예상 날씨 조회 |
| 데이터 소스 | OpenWeatherMap One Call API 3.0 (또는 기상청 단기예보 API) |
| 입력 파라미터 | `year`, `month`, `location` (기본: 서울) |
| 출력 형태 | 월 단위 캘린더 카드 — 각 날짜에 날씨 아이콘(☀️🌤️⛅🌧️❄️ 등) + 최저/최고 기온 |
| 제약 | 예보 API 한계로 14일 초과분은 기후 평년값으로 대체 표시 |

**출력 카드 예시**

```
┌──────────────────────────────────────┐
│  🗓️ 2026년 6월 날씨 (서울)            │
├───┬───┬───┬───┬───┬───┬───┤
│ 일│ 월│ 화│ 수│ 목│ 금│ 토│
├───┼───┼───┼───┼───┼───┼───┤
│   │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │
│   │☀️ │⛅ │🌧️│☀️ │🌤️│⛅ │
│   │28°│26°│24°│29°│27°│25°│
└───┴───┴───┴───┴───┴───┴───┘
```

### 4.3 SubAgent#2 — Golf

| 항목 | 내용 |
|------|------|
| 역할 | 수도권 골프장의 일자별 예약 가능 슬롯 수집 |
| 대상 지역 | 서울·경기·인천 (수도권) |
| 데이터 수집 방법 | ① 공개 API 우선 사용, ② API 미제공 시 웹 스크래핑 (robots.txt 준수) |
| 입력 파라미터 | `date_from`, `date_to`, `region`(선택), `golf_course`(선택) |
| 출력 형태 | 골프장별 예약 가능 슬롯 리스트 카드 (골프장명, 날짜, 가능 티타임 수, 가격대) |
| 수집 주기 | 요청 시 실시간 조회 (캐시 TTL: 30분) |
| 포함 골프장 (예시) | 파주·용인·광주·가평·이천 권역 주요 퍼블릭 골프장 |

**출력 카드 예시**

```
┌─────────────────────────────────────────────┐
│  ⛳ 수도권 골프장 예약 현황 (2026-06-07 기준) │
├──────────────┬──────────┬────────┬──────────┤
│ 골프장       │ 날짜     │ 슬롯   │ 가격대   │
├──────────────┼──────────┼────────┼──────────┤
│ OO컨트리클럽 │ 06/08(일)│ 3타임  │ 18~22만  │
│ △△골프장    │ 06/09(월)│ 7타임  │ 12~15만  │
│ □□리조트     │ 06/08(일)│ 1타임  │ 25만     │
└──────────────┴──────────┴────────┴──────────┘
```

### 4.4 SubAgent#3 — Schedule (A2A)

| 항목 | 내용 |
|------|------|
| 역할 | 외부 노출 MSA 목업 서버로부터 사용자 일정 조회 |
| 통신 방식 | Agent-to-Agent (A2A) — HTTP REST |
| 입력 파라미터 | `date_from`, `date_to` |
| 출력 형태 | 일자별 일정 목록 카드 |

#### Mock MSA 규칙

| 조건 | 일정 수 |
|------|---------|
| 주말 (토·일) | 2~3개 (랜덤) |
| 평일 업무 시간 내 (09:00~18:00) | 3~4개 (랜덤) |
| 하루 0건 | 가능 |
| 초과 | 가능 (드물게 더 많을 수 있음) |
| Random Seed | 고정 (`SCHEDULE_SEED=42`) — 동일 날짜 조회 시 항상 동일한 결과 반환 |

**Mock 일정 풀 (예시)**

```
- 팀 스탠드업 / 주간 회의 / 1:1 면담 / 코드 리뷰
- 점심 약속 / 외부 미팅 / 클라이언트 콜 / 기획 리뷰
- 골프 라운드 / 가족 행사 / 개인 운동 / 병원 예약
```

**출력 카드 예시**

```
┌──────────────────────────────────────────┐
│  📅 내 일정 (2026-06-07 ~ 06-08)         │
├──────────────────────────────────────────┤
│  06/07 (토)                              │
│  • 10:00  골프 라운드                    │
│  • 18:00  가족 저녁 약속                 │
│                                          │
│  06/08 (일)                              │
│  • 11:00  개인 운동                      │
└──────────────────────────────────────────┘
```

## 5. UI 명세

### 5.1 채팅 인터페이스

- **레이아웃**: 단일 페이지 채팅 (좌: 사이드바 히스토리, 우: 메시지 스트림)
- **입력**: 텍스트 입력창 + 전송 버튼
- **응답 렌더링**: 텍스트 버블 + 마크다운 카드 (HTML 테이블 또는 커스텀 카드 컴포넌트)
- **스트리밍**: SuperAgent 처리 중 스피너 표시, 결과는 카드 단위로 순차 렌더링
- **에러 상태**: 각 Sub 실패 시 부분 성공 응답 + 실패 Sub 안내 메시지

### 5.2 카드 공통 스펙

| 속성 | 내용 |
|------|------|
| 카드 타입 | `weather_calendar`, `golf_slots`, `schedule_list` |
| 헤더 | 아이콘 + 제목 + 조회 기준일 |
| 바디 | 도메인별 테이블/그리드 |
| 푸터 | 데이터 출처 + 마지막 업데이트 시각 |
